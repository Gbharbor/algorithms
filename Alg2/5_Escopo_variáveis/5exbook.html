Tipos de Passagem de Variáveis

1. Passagem de Parâmetros por Valor

O que é? Na passagem de parâmetros por valor, o procedimento recebe uma cópia dos valores das variáveis do programa principal.
Como funciona? Alterações feitas nos parâmetros dentro do procedimento não afetam as variáveis originais no programa principal.
Escopo Local: Neste exemplo, as variáveis `x` e `y` dentro do procedimento Troca têm escopo local, ou seja, elas só existem e são válidas dentro do procedimento. O que acontece com `x` e `y` dentro do procedimento não afeta as variáveis do programa principal (`num1` e `num2`).
Exemplo prático: Ao tentar trocar valores em um procedimento com passagem por valor, a troca só ocorre dentro do procedimento. No programa principal, as variáveis permanecem com os valores originais. 
algoritmo "Troca_Por_Valor"
var
    num1, num2 : inteiro

// Procedimento que tenta trocar os valores das variáveis, mas por valor
procedimento Troca(x: inteiro, y: inteiro)
var
    temp : inteiro
inicio
    temp <- x
    x <- y
    y <- temp
    escreval("Dentro do procedimento TROCA: x = ", x, " e y = ", y)
fimprocedimento

inicio
    escreva("Digite o primeiro valor (num1): ")
    leia(num1)
    escreva("Digite o segundo valor (num2): ")
    leia(num2)

    escreval("Valores antes da chamada do procedimento: num1 = ", num1, " e num2 = ", num2)
    Troca(num1, num2) // Chamada com passagem por valor
    escreval("Valores após a chamada do procedimento: num1 = ", num1, " e num2 = ", num2)

2. Passagem de Parâmetros por Referência

O que é? Na passagem de parâmetros por referência, o procedimento recebe um acesso direto às variáveis originais, em vez de uma cópia.
Como funciona? Alterações feitas nos parâmetros dentro do procedimento afetam diretamente as variáveis no programa principal.
Escopo Local e Referência: Neste exemplo, as variáveis `x` e `y` no procedimento `Troca` possuem escopo local dentro do procedimento, mas são referências diretas a `num1` e `num2` do programa principal. Qualquer modificação feita em `x` e `y` afeta `num1` e `num2` porque ambos compartilham a mesma referência.
Exemplo prático: Ao trocar valores em um procedimento com passagem por referência, a troca ocorre tanto dentro do procedimento quanto no programa principal. 
algoritmo "Troca_Por_Referencia"
var
    num1, num2 : inteiro

// Procedimento que troca os valores das variáveis usando passagem por referência
procedimento Troca(var x: inteiro, var y: inteiro)
var
    temp : inteiro
inicio
    temp <- x
    x <- y
    y <- temp
fimprocedimento

inicio
    escreva("Digite o primeiro valor (num1): ")
    leia(num1)
    escreva("Digite o segundo valor (num2): ")
    leia(num2)

    escreval("Valores antes da troca: num1 = ", num1, " e num2 = ", num2)
    Troca(num1, num2) // Chamada com passagem por referência
    escreval("Valores após a troca: num1 = ", num1, " e num2 = ", num2)

3. Uso de Variáveis Globais

O que é? Variáveis globais são declaradas no início do programa e podem ser acessadas em qualquer parte do código, inclusive dentro de procedimentos e funções.
Como funciona? Qualquer alteração feita em uma variável global dentro de um procedimento modifica o valor dela em todo o programa, pois a variável é a mesma em todos os escopos.
Escopo Global: Neste exemplo, as variáveis `num1` e `num2` são declaradas no escopo global, tornando-as acessíveis tanto no programa principal quanto dentro do procedimento `Troca`. Como são variáveis globais, qualquer modificação feita dentro do procedimento altera diretamente o valor delas no programa principal.
Exemplo prático: Ao trocar valores usando variáveis globais, a troca afeta o valor das variáveis no programa principal, já que o procedimento está usando as mesmas variáveis. 
algoritmo "Troca_Com_Variaveis_Globais"
var
    num1, num2 : inteiro // Variáveis globais

// Procedimento que troca os valores das variáveis globais
procedimento Troca()
var
    temp : inteiro
inicio
    temp <- num1
    num1 <- num2
    num2 <- temp
fimprocedimento

inicio
    escreva("Digite o primeiro valor (num1): ")
    leia(num1)
    escreva("Digite o segundo valor (num2): ")
    leia(num2)

    escreval("Valores antes da troca: num1 = ", num1, " e num2 = ", num2)
    Troca() // Chamada do procedimento que usa variáveis globais
    escreval("Valores após a troca: num1 = ", num1, " e num2 = ", num2)
fimalgoritmo

Resumo dos Três Tipos

  • Passagem por Valor: Troca ocorre apenas dentro do procedimento. O programa principal não vê nenhuma alteração. (Usa escopo local para as variáveis do procedimento)
  • Passagem por Referência: Troca ocorre dentro do procedimento e reflete no programa principal. (Usa escopo local, mas as variáveis são referências para as variáveis globais)
  • Uso de Variáveis Globais: A troca afeta todo o programa, pois as variáveis são as mesmas em todos os escopos. (Usa escopo global para as variáveis acessadas pelo procedimento)

Esses exemplos mostram como manipular variáveis em VisualG de maneira prática, dependendo do comportamento desejado para o programa.
