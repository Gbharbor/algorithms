Algoritmos II - 1 Revisao

1. O que é um algoritmo

  • Um algoritmo é uma sequência ordenada de passos para resolver um problema.
  • Definições:
    • Guimarães e Lages: Definem algoritmo como um padrão de ações bem definidas.
    • Farrer et al.: Um conjunto de comandos que leva a um resultado final.
  • Exemplo: Uma receita de bolo é uma sequência de etapas que leva a um objetivo (o bolo pronto).
  • Em programação: Algoritmos são escritos numa linguagem que o computador entende, como o Portugol, facilitando a aprendizagem.

2. Estruturas Básicas

Tipos de Dados

  • Inteiros: Números sem partes decimais (ex.: 2, -6). Declarados com inteiro.
  • Reais: Números com partes decimais (ex.: 3.5, -90.2). Declarados com real.
  • Caracteres: Texto (ex.: "A", "123"). Declarados com caractere.
  • Lógicos: Valores verdadeiro ou falso. Declarados com logico.

Operadores

  • Aritméticos: Realizam operações matemáticas.
    • Exemplos: + (soma), - (subtração), * (multiplicação), / (divisão), ** (potência).
  • Relacionais: Comparam valores, retornando verdadeiro ou falso.
    • Exemplos: = (igual), <> (diferente), > (maior), < (menor).
  • Lógicos: Combinam expressões relacionais para condições complexas.
    • Exemplos: e, ou, não.

Palavras Reservadas

  • Palavras com funções definidas na linguagem, não podem ser usadas como variáveis.
    • Exemplos: inteiro, real, caractere, logico, se, senao, enquanto, faca, para, ate, leia, escreva.

Estrutura Básica de um Algoritmo

  1. Declaração do Algoritmo: 
    algoritmo "Nome do Programa"
  2. Comentários: Explicações no código, iniciadas com //.
  3. Declaração de Variáveis: 
    var idade : inteiro nome : caractere
  4. Bloco Principal: Iniciado com inicio e finalizado com fimalgoritmo. 
    inicio escreva("Olá, mundo!") fimalgoritmo

3. Comandos Básicos

  1. Atribuição: Usa <- para definir valores em variáveis. 
    x <- 10
  2. Leitura (leia): Lê entrada do usuário. 
    leia(x)
  3. Escrita (escreva): Exibe valores ou mensagens na tela. 
    escreva("Valor de x:", x)
  4. Comentários: Explicações no código. 
    // Comentário de linha única { Comentário em bloco }

4. Estruturas Condicionais e de Repetição

Condicionais

  1. se...entao...senao: Executa blocos com base em uma condição. 
    se media >= 7.0 entao escreva("Aprovado!") senao escreva("Reprovado!") fimse
  2. Aninhamento de Condicionais: Para verificações complexas. 
    se idade >= 18 entao escreva("Maior de idade") senao se idade >= 0 entao escreva("Menor de idade") senao escreva("Idade inválida") fimse fimse

Repetições

  1. Enquanto...faça: Repete enquanto uma condição for verdadeira. 
    enquanto x < 10 faca escreva(x) x <- x + 1 fimenquanto
  2. Repita...até: Repete pelo menos uma vez até a condição ser verdadeira. 
    repita escreva("Digite um número menor que 100:") leia(x) ate x < 100
  3. Para...até...faça: Repetição com contagem determinada. 
    para i de 1 ate 5 faca escreva(i) fimpara

5. Vetores e Matrizes

Vetores

  • Sequência de dados do mesmo tipo, acessada por índices. 
    idades : vetor[1..10] de inteiro
  • Exemplo de uso: idades[3] <- 18.

Matrizes

  • Tabela bidimensional, com linhas e colunas. 
    notas : vetor[1..5, 1..3] de real
  • Exemplo de acesso: media <- notas[1][2].
