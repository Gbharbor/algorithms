Algoritmos II - 7 Arquivos

1. Conceito de Arquivo em Algoritmos

Um arquivo é um conjunto de registros que permite armazenar dados permanentemente, mesmo após o encerramento do programa. Diferente de variáveis, que são temporárias, arquivos gravam informações em memória secundária (HD, SSD) e garantem que os dados possam ser acessados futuramente.

Exemplo de Arquivo

Imagine uma agenda telefônica. Usando arquivos, o programa pode gravar contatos como nome e número de telefone. Quando o programa é reaberto, esses dados ainda estarão disponíveis para consulta ou edição.

2. Declaração e Abertura de Arquivos

Para trabalhar com um arquivo em algoritmos, devemos primeiro declarar e abrir o arquivo.

  • Declaração: Declara-se um arquivo no bloco var com a seguinte estrutura:
    • <variavel> : arquivo de <tipo_registro>

    Aqui, <tipo_registro> define o tipo de dados que o arquivo armazenará, semelhante à criação de um tipo de variável.

  • Exemplo:
    • var
AGENDA : arquivo de TipoContato
  • Abertura: Após a declaração, o arquivo é aberto em um dos seguintes modos:
    • leitura: permite acessar dados existentes.
    • escrita: permite adicionar novos dados ao final do arquivo.

    Exemplos:

    abra AGENDA leitura
abra AGENDA escrita

    Importante: Abrir e Fechar Arquivos

    O arquivo deve ser sempre fechado após seu uso para garantir que todos os dados sejam gravados e o acesso ao arquivo seja liberado. Para isso, usamos a instrução feche <variavel>.

    feche AGENDA

3. Manipulação de Ponteiros

Ponteiro: Um ponteiro é a posição atual de leitura ou escrita no arquivo.

  • No modo leitura, o ponteiro começa no início.
  • No modo escrita, o ponteiro começa no final, para adicionar novos registros sem apagar os existentes.
  • EOF (End Of File): A função <variável>.EOF verifica se o final do arquivo foi atingido, evitando que o programa tente ler além dos dados disponíveis.

Exemplo Usando EOF

enquanto .NÃO. AGENDA.EOF faca
AGENDA.leia(contato)
escreva("Nome: ", contato.nome, " - Telefone: ", contato.telefone)
fimenquanto

4. Leitura de Arquivos

Para ler dados de um arquivo, a função <variável>.leia(<variável_do_tipo_do_arquivo>) é usada. A leitura é sequencial, ou seja, os dados são lidos na ordem em que foram gravados.

Exemplo de Leitura

var
AGENDA : arquivo de TipoContato
contato : TipoContato
inicio
    abra AGENDA leitura
    enquanto .NÃO. AGENDA.EOF faca
        AGENDA.leia(contato)
        escreva("Nome: ", contato.nome, " - Telefone: ", contato.telefone)
    fimenquanto
    feche AGENDA
fimalgoritmo

5. Escrita de Arquivos

A escrita permite gravar novos dados ao final do arquivo, usando <variável>.escreva(<variável_do_tipo_do_arquivo>).

Exemplo de Escrita

var
AGENDA : arquivo de TipoContato
contato : TipoContato
inicio
    contato.nome <- "Alice"
    contato.telefone <- "1234-5678"
    
    abra AGENDA escrita
    AGENDA.escreva(contato)
    feche AGENDA
fimalgoritmo

6. Benefícios e Cuidados com o Uso de Arquivos

  • Persistência de Dados: Arquivos garantem que os dados fiquem salvos, mesmo após o fechamento do programa.
  • Gerenciamento de Ponteiros: Controle o ponteiro para ler ou escrever na posição correta.
  • Abertura e Fechamento Cuidadoso: Sempre abrir e fechar os arquivos corretamente evita erros de gravação e liberação de recursos.
  • EOF: Utilize o EOF para assegurar que o programa não tente ler além do final do arquivo.

Exemplo Completo: Manipulação de Arquivo de Contatos

tipo
TipoContato = registro
    nome : caractere
    telefone : caractere
fimregistro

var
AGENDA : arquivo de TipoContato
contato : TipoContato
opcao : inteiro

inicio
    escreva("Digite 1 para listar contatos ou 2 para adicionar um novo contato: ")
    leia(opcao)

    se (opcao = 1) entao
        abra AGENDA leitura
        enquanto .NÃO. AGENDA.EOF faca
            AGENDA.leia(contato)
            escreva("Nome: ", contato.nome, " - Telefone: ", contato.telefone)
        fimenquanto
        feche AGENDA
    senao
        se (opcao = 2) entao
            escreva("Digite o nome do contato: ")
            leia(contato.nome)
            escreva("Digite o telefone do contato: ")
            leia(contato.telefone)
            
            abra AGENDA escrita
            AGENDA.escreva(contato)
            feche AGENDA
        fimse
    fimse
fimalgoritmo

Resumo Final

  • Declaração e Abertura: Arquivos precisam ser declarados e abertos em leitura ou escrita.
  • Ponteiros e EOF: Controlam a posição de leitura e evitam leitura excessiva.
  • Leitura e Escrita: Manipulam dados de forma sequencial e cuidadosa.
  • Fechamento: Sempre feche o arquivo após o uso.

Compreender a manipulação de arquivos permite que dados sejam armazenados de forma persistente, proporcionando mais funcionalidade e flexibilidade ao programa.
