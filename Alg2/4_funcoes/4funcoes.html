Algoritmos II - 4 Functions

Funções são blocos de código que realizam uma tarefa específica e retornam um valor para o bloco de código que as chamou. Elas são ideais para tarefas que exigem um cálculo ou processamento cujo resultado é necessário para continuar a execução do programa.

Estrutura de uma Função

funcao <nome>(<parâmetros>): <tipo de dado>
var
    <variáveis locais>
inicio
    <instruções>
    retorne(<valor>)
fim
  • <nome>: Nome da função, usado para chamá-la posteriormente.
  • <parâmetros>: Variáveis que a função recebe para realizar a tarefa. Parâmetros são opcionais.
  • <tipo de dado>: Define o tipo de dado que a função retornará (ex.: inteiro, real, logico, etc.).
  • variáveis locais: Variáveis usadas apenas dentro da função.
  • retorne(<valor>): Encerra a função e passa o valor de retorno para o ponto onde a função foi chamada.

Diferença entre Procedimento e Função

  • Procedimento: Executa uma tarefa específica, como exibir uma mensagem, mas não retorna um valor.
  • Função: Executa uma tarefa e retorna um valor, que pode ser utilizado em expressões ou atribuições.

Exemplo Comparativo

procedimento ESCREVER_SOMA(A: inteiro, B: inteiro)
var
    resultado: inteiro
inicio
    resultado <- A + B
    escreva("Resultado da soma: ", resultado)
fim

funcao SOMA(A: inteiro, B: inteiro): inteiro
var
    resultado: inteiro
inicio
    resultado <- A + B
    retorne(resultado)
fim

No programa principal, podemos usar a função SOMA em uma expressão:

algoritmo "Exemplo de Uso de Função"
var
    num1, num2, resultado: inteiro
inicio
    leia(num1)
    leia(num2)
    resultado <- SOMA(num1, num2) + 10 // Uso de função em uma expressão
    escreva("Resultado final: ", resultado)

Aplicação Prática de Funções

1. Função para Cálculo de Reajuste Salarial

Exemplo de função para calcular o novo salário com base no salário atual e nas faixas de reajuste.

funcao REAJUSTE_SALARIAL(salario: real): real
var
    novo_salario: real
inicio
    se salario <= 2000 entao
        novo_salario <- salario * 1.10
    senao
        se salario <= 5000 entao
            novo_salario <- salario * 1.07
        senao
            novo_salario <- salario * 1.05
        fimse
    fimse
    retorne(novo_salario)
fim

Uso no programa principal:

algoritmo "Calculo de Reajuste"
var
    salario_atual, salario_novo: real
inicio
    escreva("Digite o salário atual: ")
    leia(salario_atual)
    salario_novo <- REAJUSTE_SALARIAL(salario_atual)
    escreva("Salário reajustado: ", salario_novo)
fimalgoritmo

2. Função para Verificação de Maioridade

Função que retorna verdadeiro se a pessoa for maior de idade e falso caso contrário.

funcao E_MAIOR_IDADE(idade: inteiro): logico
inicio
    se idade >= 18 entao
        retorne(verdadeiro)
    senao
        retorne(falso)
    fimse
fim

Uso no programa principal:

algoritmo "Verificacao de Maioridade"
var
    idade: inteiro
inicio
    escreva("Digite a idade: ")
    leia(idade)

    se E_MAIOR_IDADE(idade) entao
        escreva("A pessoa é maior de idade.")
    senao
        escreva("A pessoa é menor de idade.")
    fimse
fimalgoritmo

Escopo de Variáveis em Funções

Variáveis declaradas dentro de uma função possuem escopo local, acessíveis apenas dentro da função onde foram declaradas, evitando conflitos com variáveis externas.

funcao CALCULA_QUADRADO(numero: inteiro): inteiro
var
    resultado: inteiro
inicio
    resultado <- numero * numero
    retorne(resultado)
fim

Resumo

Funções permitem modularizar o código, criando blocos que realizam tarefas específicas e retornam valores. As principais vantagens incluem:

  • Reutilização de código: Podemos usar a mesma função várias vezes.
  • Organização e clareza: Dividem o programa em blocos menores.
  • Escopo local: Variáveis dentro das funções não interferem em outras partes do programa.
